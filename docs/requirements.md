# Requirements Document
## User Stories: 
1) As a new PhD student, I want to be able to easily navigate this reference guide/website, so that I don’t get overwhelmed.
2) As a PhD student, I want to see all available data sources, so I can select the best source to collect the data I’m looking for.
3) As a researcher, I want to compare collection methods, so I can choose the most appropriate method for my type of application.
4) As an analyst, I want to reference many types of analysis, so I can best analyze the data I’m working with.
5) As a researcher, I want to see all publication venues, so I can decide through which to most effectively publish my research.

## Functional Requirements:
-	simple web page
-	four main sections:
o	Data Sources
  - Source name
  - What kind of data it provides
  - Brief description
o	Collection Methods
  - Method name
  - What it’s used for
  -	Pros and cons
o	Analysis Methods
  -	Method name
  -	Type of analysis
  -	Common use cases
o	Publication Venues
  -	Venue name and abbreviation
  -	Type (conference or journal)
  -	Focus area
  -	How relevant to MSR (high, medium)
-	Simple tabs or buttons to switch sections
-	Simple table or list format

## Non-Functional Requirement: 
-	Not-over whelming to use
-	Clean and readable
-	Professional looking

## Constraints: 
-	Must be a web page

## Assumptions:
-	Assuming Windows computers
